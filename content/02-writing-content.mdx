# Writing Content

veslx uses MDX - markdown that supports JSX. Write markdown or embed React components.

## Frontmatter

Pages start with YAML frontmatter:

```yaml
---
title: "Page Title"
description: "Brief description for listings"
date: "2025-01-15"
draft: true           # Hide from listings
visibility: "hidden"  # Also hides from listings
---
```

Display it with `<FrontMatter />`:

```mdx
<FrontMatter />
```

## File Ordering

Files sort by:

1. **Numeric prefix** - `01-intro.mdx` sorts first, displays as "Intro"
2. **Date** - Files with dates sort newest first
3. **Alphabetical** - Everything else

```
docs/
├── 01-getting-started.mdx  → "Getting Started" (1st)
├── 02-configuration.mdx    → "Configuration" (2nd)
├── my-post.mdx             → Has date, sorts by date
└── zebra.mdx               → Alphabetical
```

## Posts vs Docs

Content is classified based on the `date` frontmatter field:

- **Posts** - Content with a `date` field (blog posts, articles, changelog entries)
- **Docs** - Content without a `date` field (documentation, guides, reference)

### Direct URLs

Filter content by visiting:

- `/posts` - Show only dated content
- `/docs` - Show only undated content
- `/` - Show all content (or configured default)

### Default View

Set the default view in `veslx.yaml`:

```yaml
site:
  name: My Site
  defaultView: posts  # or 'docs' or 'all'
```

When both posts and docs exist, tabs appear on the homepage to switch views.
