# Configuration

veslx works without config. Values are derived from `package.json` and git.

## Zero Config Defaults

- `name` → from `package.json` or folder name
- `description` → from `package.json`
- `github` → parsed from git remote
- `defaultView` → "all"

## Optional Config

Create `veslx.yaml` to override:

```yaml
dir: docs
site:
  name: My Project
  description: Project documentation
  github: user/repo
  defaultView: docs
```

## Generate Config

```bash
veslx init
```

Creates a `veslx.yaml` with detected values.

## Options

### `dir`

Content directory relative to config file.

```yaml
dir: docs
```

Default: `.` (current directory)

### `site.name`

Site name shown in header and browser title.

```yaml
site:
  name: My Docs
```

Default: `package.json` name or folder name

### `site.description`

Homepage subtitle.

```yaml
site:
  description: Documentation for my project
```

Default: `package.json` description

### `site.github`

GitHub repository link shown in header.

```yaml
site:
  github: user/repo
```

Default: parsed from `git remote get-url origin`

### `site.defaultView`

Default content view on homepage.

```yaml
site:
  defaultView: docs
```

Options:
- `all` - Show all content (default)
- `docs` - Show only undated content
- `posts` - Show only dated content

## Navigation

The header includes:

- **Site name** - Links to homepage
- **Posts** - Filter to dated content
- **Docs** - Filter to undated content
- **GitHub** - Link to repository (if configured)
- **Theme toggle** - Dark/light mode

## Content Views

Navigate directly to filtered views:

- `/` - Default view (configured via `defaultView`)
- `/posts` - Only content with `date` frontmatter
- `/docs` - Only content without `date` frontmatter
- `/all` - All content

## Example Config

```yaml
dir: content
site:
  name: My Project
  description: Technical documentation and blog
  github: myorg/myproject
  defaultView: docs
```
