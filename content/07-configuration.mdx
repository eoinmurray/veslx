# Configuration

veslx works without config. Values are derived from `package.json` and git.

## Zero Config Defaults

- `name` → from folder name
- `description` → from `package.json`
- `github` → parsed from git remote
- `defaultView` → "all"

## Optional Config

Create `veslx.yaml` to override:

```yaml
dir: docs
site:
  name: My Project
  description: Project documentation
  github: user/repo
  defaultView: docs
```

## Generate Config

```bash
veslx init
```

Creates a `veslx.yaml` with detected values.

## Options

### `dir`

Content directory relative to config file.

```yaml
dir: docs
```

Default: `.` (current directory)

### `site.name`

Site name shown in header and browser title.

```yaml
site:
  name: My Docs
```

Default: `package.json` name or folder name

### `site.description`

Homepage subtitle.

```yaml
site:
  description: Documentation for my project
```

Default: `package.json` description

### `site.github`

GitHub repository link shown in header.

```yaml
site:
  github: user/repo
```

Default: parsed from `git remote get-url origin`

### `site.defaultView`

Default content view on homepage.

```yaml
site:
  defaultView: docs
```

Options:
- `all` - Show all content (default)
- `docs` - Show only undated content
- `posts` - Show only dated content

### `slides.scrollSnap`

Enable or disable scroll snapping for slide presentations.

```yaml
slides:
  scrollSnap: true
```

When enabled, scrolling will snap to each slide. Set to `false` for free scrolling.

Default: `true`

## Navigation

The header includes:

- **Site name** - Links to homepage
- **Posts** - Filter to dated content
- **Docs** - Filter to undated content
- **GitHub** - Link to repository (if configured)
- **Theme toggle** - Dark/light mode

## Content Views

Navigate directly to filtered views:

- `/` - Default view (configured via `defaultView`)
- `/posts` - Only content with `date` frontmatter
- `/docs` - Only content without `date` frontmatter
- `/all` - All content

## Example Config

```yaml
dir: content
site:
  name: My Project
  description: Technical documentation and blog
  github: myorg/myproject
  defaultView: docs
```

## LLM Access

veslx generates an `llms.txt` file during build that makes your content accessible to CLI tools and LLMs. This file lists all MDX content with links to raw source files.

Access at `/llms.txt` on your deployed site:

```
# My Project

Project description here.

> This is a veslx documentation site. Content is served as raw MDX files.

## Content

- [doc] Getting Started: /raw/getting-started.mdx
- [slides] My Presentation: /raw/talk/SLIDES.mdx
```

Raw MDX files are served at `/raw/{path}` and can be fetched directly.
